Yaml files

Multiline documentation

Yaml only allows single-line comments. However, it is possible to add multiline content to a Yaml data element.

If the application that will use the yaml file ignores unknown data elements, we can add our own "yamldoc". The name of the element is arbitrary. We use _yamldoc to make its purpose clear.

Adding our own data element will not work if an applications uses a strictly defined yaml schema.
The name _yamldoc is arbitrary. To add multiple sections under one Yaml key, each name must be unique, so simply use _yamldoc1, _yamldoc2, …​ for the next instances. 
element1:
  attribute1: ...
  _yamldoc: >
    comments (1)
    comments
  attribute2: ... (2)
1 Start the multiline value with the element name and >. The next lines have to be indented.
2 End of multiline value when the indentation ends.

Since the multiline comments are indented, we need to add the parameter indent=0 when including the tagged area in the documentation.

Example

The example below is based on a section of our Antora playbook for the documentation site.

In the yaml file to be documented
# tag::asciidoc[] (1)
asciidoc:
  extensions:
  - asciidoctor-kroki # <1> (2)
  attributes:
    page-pagination: true
    hide-uri-scheme: true # <2> (3)
    kroki-fetch-diagram: true # <3> (4)
    allow-uri-read: true
    experimental: true
    kroki-default-format@: svg
    kroki-server-url: http://localhost:18181 # <4> (5)
# end::asciidoc[] (6)
    _yamldoc: >
      tag::asciidoc-notes[] (7)
      <1> Use kroki to generate diagram images
      <2> Don't show https:// etc on bare links
      <3> For kroki: store the image. This requires `allow-uri-read`, as well as `experimental` (to override kroki security)
      <4> To run  a local kroki server: `docker run -p18181:8000 -d --name asciidoctor-kroki yuzutech/kroki`
      end::asciidoc-notes[] (8)
1 Start of the source to be included in the documentation.
2 Specify a callout
3 idem
4 idem
5 idem
6 End of the source to be included.
7 Start the notes section as _yamldoc element, with the text for the callouts.
8 End of the notes section, and of the multiline element.
In the documentation source

[source, yaml]
.My documented configuration
----
include::../examples/my-documented-configuration.yml[tags=asciidoc] (1)
----
include::../examples/my-documented-configuration.yml[tags=asciidoc-notes, indent=0] (2)

Rest of the documentation page ...
1 Include the script portion in a code block.
2 Include the callouts text as regular Asciidoc after the code block. Use indent=0 to remove leading whitespace.
